Instantiate the Above The Genes Organizational Singularity as an incremental, contest-safe Guardian Mosaic product line that can win XPRIZE evidence review while preserving the deeper private canon for later regulated and enterprise expansion.

## Public Product Lane

The public app remains observe-and-dialogue wellness support.

Allowed:
- Apple Health and wearable trend intake
- subjective check-ins
- sleep, recovery, activity, HRV, respiration, temperature, EDA, adherence, and reflection trends
- daily wellness routing
- purpose, flow, recovery, breathwork, and reflection prompts
- beta intake, pilot evidence, aggregate reporting, and sponsor dashboards

Disallowed in the public repo/app:
- diagnosis
- treatment claims
- medication advice
- autonomous clinical decisions
- fitness-for-duty or return-to-duty decisions
- private PHI exposure
- proprietary internal biological design content

## Private Canon Lane

The private canon may retain deeper invention language, corpus structure, Mosaic Organism architecture, GraST CCF, GFIE, 6D tesseract, dense cube, lottery ticket, and research lineage. Public artifacts must expose only contest-safe abstractions and receipts.

## Execution Lanes

1. Canon Lock
   - identify latest canonical files
   - hash and receipt each canonical source
   - produce a promoted canon index

2. Corpus Pipeline
   - inventory local, iCloud, Google Drive, Dropbox, SSD, and repo sources
   - normalize PDFs, DOCX, TXT, Markdown, CSV, JSON, screenshots, and transcripts
   - deduplicate, classify, chunk, and provenance-tag
   - promote only clean, source-linked units

3. Mosaic Organism Optimization
   - refresh awareness and continuity payloads
   - enforce partner experience gates
   - run gameplay/regression loops for Hermes and `/goal`
   - require fresh accountability telemetry before promotion

4. Sensorium Implementation
   - start with Apple Health export/manual import
   - add CSV/JSON adapters for Garmin, Oura, Eight Sleep, Withings, iHealth, Frontier X, Peloton, Concept2, sauna, and red-light sessions
   - normalize all data as low-confidence wellness trends

5. App Delivery
   - keep iOS TestFlight moving first
   - add founder `/goal` and daily check-in flows
   - integrate local corpus-backed guidance after canon lock
   - capture screenshots and demo video evidence

6. Google/XPRIZE Evidence
   - enforce `docs/xprize-rules-governance.md`
   - maintain Google/Workspace evidence artifacts
   - run `scripts/google_ops_runner.py`
   - run `scripts/doc_strange_adversarial_loop.py`
   - keep judge-facing repo current and safe

7. Factory/Bridge Discipline
   - no duplicate assignments
   - no orphaned bridge windows
   - no stale accountability receipts
   - no yellow/red promotion gates
   - prefer local deterministic work before model calls

## Green Gates

An artifact is green only when:
- source lineage is recorded
- safety boundary is explicit
- implementation status is current
- tests or validation receipts exist
- XPRIZE relevance is clear
- no stale bridge/task state is blocking progress
