# Goal Routing

## Purpose

Translate a high-level operator goal into a small number of contest-safe execution steps for the `ATG Responder Coherence Operator`.

## Routing Principles

- Keep goals tied to the public-safety wellness wedge.
- Reduce each goal to a minimal set of actionable steps.
- Use local and Workspace-native work first.
- Prefer evidence capture over speculative features.
- Do not surface proprietary methods, internal model details, or private data in the public repo.

## Goal Intake

Accept goals only if they support one of these outcomes:
- contest compliance
- public demo quality
- pilot readiness
- revenue evidence
- user evidence
- safety and privacy compliance
- future regulated-track planning, if and only if it remains private and out of the public repo

## Goal Decomposition

Each goal should break into:
- one user-facing outcome
- one evidence artifact
- one safety boundary
- one minimal implementation step

## Execution Order

1. verify scope and safety
1. choose the minimum viable implementation path
1. use existing subscriptions and owned data first
1. capture receipts, logs, and screenshots
1. stop when the goal is demonstrably satisfied

## Stop Conditions

Stop the loop when:
- the public artifact is contest-safe
- the demo still works
- the IP boundary is intact
- the additional work would add cost or exposure without improving submission quality
- the requested work would move the public demo into regulated clinical territory

## Result

This is the public-facing goal system for the XPRIZE submission: small, bounded, measurable, and safe.
